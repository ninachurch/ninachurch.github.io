The Spiel on Rails

  • Phase 0: Week 9

Rails is a web application framework. Rails is built on top of the ruby programming language.

Before we go any further, lets check that you understand the prerequisites. A programming language is a type of computer language that allows you to instruct a computer program how to act. A web application is a site you visit on the web that you can interact with, like Google.com, or Facebook.com.

It’s also important to note that when a website is visited, it is delivered to the browser in HTML language. You cannot send programs written in ruby straight to a browser! They won’t understand the ruby, however they do understand HTML.

Imagine that you want to create a web application (an interactive website), and that you want to do it using ruby. You would have to program many basic and commonly repeated pieces, such as programs for how to tell which page a user is trying to visit, how to receive data that a user enters on a web page, and so on.

Web application frameworks take many of these commonly repeated pieces and package them up into a framework, making it faster to build a web application. They can also enforce common standards, so that all the people using the framework write web applications in a similar way, making it easier to work together with others on the same application, or to upgrade old applications.

Therefore, rails, or ruby on rails, is a web application framework that makes it simpler and faster to build web applications using the ruby programming language.
