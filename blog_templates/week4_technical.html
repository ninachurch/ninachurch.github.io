Ruby vs. Javascript

A Guide on Choosing the Right Language for You

  • Phase 0: Week 4
The Lowdown

Whether you're a complete newbie or a knee deep programmer, you've likely heard of both Ruby and Javascript. Use this guide to help you decide which language to go with!

Ruby
Pros

Ruby is an intuitive programming language. Both JavaScript and Ruby are dynamically typed, meaning that you don't need descriptors that indicate what type of information you're putting into every method. However Ruby is often rated as a more friendly syntax than JavaScript.

Cons

Many people would still consider Ruby a "fad" that will pass soon. People say that to make Ruby as intuitive as it is, there are many "layers of abstraction" that seperate the programmer from what's actually happening, which can provide security and other sub optimal issues.

JavaScript
Pros

JavaScript is "the Browser of the Web" because it allows you to work on the backend and integrate seamlessly into the front-end. This is incredibly consistent and convinient. It is also growing in popularity, and speed now that Google's V8 Javascript Engine exists.

Cons

There are many weird idiosyncratic things that the language does, in particular in syntax. It also doesn't allow you to simplify quite as much as Ruby.
